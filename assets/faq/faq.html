1. WHAT IS FLOREANTPOS

FloreantPOS is an opensource initiative to provide enterprise class of point of sales to restaurants or cafes.

2. HOW IT WORKS?

FloreantPOS for Android works by accessing a web interface via WiFi connection. The web interface access database of FloreantPOS that also used by FloreantPOS Desktop version. By this design, the database of FloreantPOS must support multi-sessions such as Derby Server, MySQL, Oracle, etc. Derby Single as default database of FloreantPOS can't be used.

Unless your database is hosted on internet, your Android phone must support WiFi to run this application.

3. WHAT IS THE BEST PRACTICE TO USE FLOREANTPOS ANDROID?

FloreantPOS for Android is used as compliment to Desktop version of FloreantPOS. Menu related activities performed by restaurant servants using FloreantPOS. Payment related could only be done on Desktop version.

4. WILL IT PRINT ORDER TO KITCHEN PRINT?

Yes it will prints ticket details into kitchen printer if configured so.

5. HOW TO INSTALL THE SERVER COMPONENT of FLOREANTPOS ANDROID?

5.1. Install Tomcat Web Server

- Download Tomcat from www.apache.org (Choose version 7.0 or above)

- Unzip the tomcat into C:/FloreantPos/tomcat

5.2. Install Java Runtime

- Download JRE 1.7 or 1.8 from http://www.oracle.com

- Install JRE into C:/FloreantPOS/jre

5.3 Download Database JDBC Driver

- Download JDBC driver of derby server from http://www.eu.apache.org/dist//db/derby/db-derby-10.11.1.1/db-derby-10.11.1.1-bin.zip

- Copy the jars into C:/FloreantPOS/tomcat/lib/

5.3. Configure JDBC JNDI at Tomcat Configuration

In conf folder of Tomcat, there exist a context.xml file. This file requires additional xml tag of Resource; please specify the appropriate IP Address, Port, Database Name, User and Password of database.

Tomcat/conf/context.xml

<?xml version='1.0' encoding='utf-8'?>

<Context>

<WatchedResource>WEB-INF/web.xml</WatchedResource>

<Resource name="jdbc/FloreantPOSDB"

auth="Container"

type="javax.sql.DataSource"

maxActive="100"

maxIdle="30"

maxWait="10000"

username="app"

password="sa"

driverClassName="org.apache.derby.jdbc.ClientDriver"

url="jdbc:derby://localhost:1527/posdb"/>

</Context>

 

5.4. Download floreant.war from www.indocert.com/floreantposv14/floreant.war and put it into folder C:/FloreantPos/tomcat/webapps

5.5. Start Tomcat up by executing C:/FloreantPos/tomcat/bin/startup.bat (or startup.sh for linux)

 

6. WHAT IS THE DEFAULT USER IN FLOREANTPOS?

FloreantPOS by default add an admin user with secret key 7777
